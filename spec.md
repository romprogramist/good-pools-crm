# CRM «Хорошие Бассейны» — Спецификация

> Версия 1.0 — 2026-05-08
> Статус: утверждено клиентом

---

## 1. Бизнес-контекст

CRM-система для компании **«Хорошие Бассейны»**, обслуживающей частные бассейны.

- **Сейчас:** одна компания, ~10 сервисников, от 80 клиентов и больше.
- **В перспективе:** возможна продажа системы другим компаниям → каждой компании отдельная установка (multi-tenant НЕ закладываем).
- **База:** стартуем с пустой, импорт не нужен.

## 3. Технологический стэк

| Слой | Решение |
|---|---|
| Фреймворк | Next.js 16 (App Router) + TypeScript |
| БД | PostgreSQL + Prisma ORM |
| UI | Tailwind CSS + shadcn/ui |
| Auth | Auth.js (NextAuth) — Credentials provider, сессии в БД |
| Файлы | Локально в `/uploads/` + защищённый роут `/api/files/[...]` с проверкой прав |
| PDF | `@react-pdf/renderer` (без Puppeteer) |
| Карты | Яндекс.Карты JS API v3 |
| Push | Web Push API + VAPID, Service Worker (Serwist) |
| PWA | Serwist — манифест + offline-shell |
| Чат | Поллинг каждые 5 секунд при открытом чате |
| Cron | `node-cron` в отдельном worker-процессе |
| Деплой | Docker Compose: postgres + app + worker + nginx + LetsEncrypt |

## 4. Роли и права

### Админ
- Полный доступ ко всем данным.
- Заводит сервисников и клиентов (отправка email-приглашений).
- Редактирует вопросы чек-листа.
- Редактирует прайс на химию.
- Видит чат клиента с поддержкой.
- Может работать как сервисник (заполнять чек-листы).
- Видит реестры, журнал действий, экспортирует в Excel/CSV.

### Сервисник
- Видит всех клиентов и их бассейны.
- Создаёт и дополняет карточки клиентов и бассейнов.
- Ставит визит в календаре, видит визиты других сервисников.
- Принимает онлайн-заявки клиентов (назначает диапазон дат).
- Заполняет чек-лист на объекте, прикрепляет фото.
- Добавляет в счёт химию (из прайса) и доп. работы (вручную).
- Обновляет счётчик регламента после замены детали.
- Видит чат клиента с поддержкой и отвечает.
- За конкретным бассейном НЕ закреплён — приехать может любой свободный.

### Клиент
- Регистрируется по email-приглашению от админа.
- Видит свою карточку и карточки своих бассейнов.
- Видит историю обслуживаний.
- Читает инструкции.
- Отправляет онлайн-заявку на сервис (без выбора конкретного слота — указывает желаемый диапазон).
- Не может отменить онлайн-заявку.
- Видит PDF-отчёт после визита.
- Пишет в чат поддержки, прикрепляет фото.
- Видит сумму к оплате после визита; возможна онлайн-оплата (когда подключим провайдер).
- При наличии долга — не может отправить новую онлайн-заявку.

## 5. Высокоуровневая модель данных

```
User (id, email, password_hash, role, name, phone, created_at)
  ├── role: admin | service | client
  └── связь client → Customer (1:1)

Customer (id, user_id, full_name, phone, email, legal_info?, individual_service_price)

Pool (id, customer_id, name, address, lat, lng, photos[], facing_materials, extra_field, instructions[])
  ├── Equipment (id, pool_id, type_id, serial?, install_date, warranty_months, last_replacement_date)
  └── EquipmentTemplate (id, type_name, default_warranty_months, regulation_period_days)

Visit (id, pool_id, service_user_id, scheduled_at, status, kind: planned|online_request|completed|...)
  ├── ChecklistAnswer (id, visit_id, question_id, value)
  ├── VisitPhoto (id, visit_id, path, uploaded_at)
  ├── VisitChemistry (id, visit_id, chemistry_id, qty, price_at_moment)
  ├── VisitExtraWork (id, visit_id, name, price)
  └── PaymentStatus (paid | unpaid | online_pending)

OnlineRequest (id, customer_id, pool_id, desired_from, desired_to, message?, accepted_visit_id?, status)

ChecklistQuestion (id, order, type: text|number|select|bool, label, options?[], active)
ChemistryItem (id, name, unit, price, active)

VisitSeries (id, pool_id, recurrence_rule, ...)  ← для повторяющихся визитов

ChatThread (id, customer_id)
  └── ChatMessage (id, thread_id, sender_user_id, body, photo_path?, created_at)

Notification (id, user_id, type, title, body, payload, read_at?, created_at)
PushSubscription (id, user_id, endpoint, keys, user_agent, created_at)

ActivityLog (id, actor_user_id, action, entity_type, entity_id, diff?, created_at)
```

Имена полей — ориентир, финальная схема в Prisma при реализации.

## 6. Хранение файлов

- Папка `uploads/` на сервере, разбита по сущностям: `uploads/pool-photos/{pool_id}/`, `uploads/visit-photos/{visit_id}/`, `uploads/chat/{thread_id}/`, `uploads/reports-pdf/{visit_id}/`.
- Доступ только через `/api/files/...` с проверкой прав (клиент видит только свои файлы; сервисник/админ — всё).
- Cron-задача чистит фото визитов и чата старше **3 месяцев**.
- PDF-отчёты хранятся бесконечно (фото внутри PDF могут оставаться даже после удаления исходных файлов).
- Текстовые данные (карточки, ответы чек-листов, журналы) — хранятся бесконечно.

## 7. Уведомления

| Событие | Кому | Канал | Когда |
|---|---|---|---|
| Новая онлайн-заявка от клиента | Всем сервисникам + админу | Push | Сразу |
| Онлайн-заявка принята сервисником | Клиенту | Push | Сразу |
| Окончание гарантии оборудования | Клиенту | Push | За 14 дней |
| Наступление регламента замены | Клиенту + всем сервисникам | Push | За 7 дней |
| Новое сообщение в чате (от клиента) | Всем сервисникам + админу | Push | Сразу |
| Новое сообщение в чате (от поддержки) | Клиенту | Push | Сразу |
| PDF-отчёт готов | Клиенту | Push + email | Сразу после завершения визита |

Push требует:
- HTTPS (продакшн),
- VAPID-ключи в env,
- Клиент один раз даёт permission в браузере → подписка сохраняется в `PushSubscription`.

## 8. Деплой

```
[ nginx (LetsEncrypt) ]
       │
   ┌───┴────┐
   │  app   │  ← Next.js (Node)
   └───┬────┘
       │
   ┌───┴──────┐    ┌──────────┐
   │ postgres │    │  worker  │  ← node-cron + рассылка push
   └──────────┘    └──────────┘

Все в docker-compose.yml на сервере клиента.
Бэкап postgres → ежедневно в /backups/, ротация 30 дней.
Папка uploads/ → монтируется как volume.
```

## 9. Что НЕ входит в первую версию (явно)

- Multi-tenancy.
- Импорт из других систем.
- 2FA.
- SMS-уведомления.
- Учёт остатков химии на складе.
- История изменений индивидуальной цены сервиса.
- Подключение конкретного платёжного провайдера (готовим интерфейс, выбор откладываем).
- Мобильные приложения (iOS/Android native) — только PWA.
